Тип scene (Сценарий)
====================

Сценарный блок создаётся в основном меню и используется для активации сценариев или режимов пользователем. При нажатии на блок панель отправляет соответствующее сообщение в указанный MQTT-топик. 
Параметры:

- :ref:`scene-param_1` — назначение блока (например, тип действия).
- :ref:`scene-param_2` — локация или контекст использования.
- :ref:`scene-param_3` — название сценария или режима.
- :ref:`scene-icon` — иконка, отображаемая на блоке.
- :ref:`scene-command_topic` — MQTT-топик для отправки команды.
- :ref:`scene-payload` — команда, отправляемая при нажатии.


Детальный разбор параметров:
----------------------------

.. _scene-param_1:
param_1
*****
   Используется для изменения текста в UI-блоке, например, чтобы указать, что это «Сценарий» или «Режим». Может применяться и в других целях.

   :имя поля: ``param_1``
   :тип объекта: ``String``
   :наличие значения: Может быть пустым
   :ограничение строки: 9 символов
   :пример-1: ``"param_1":"Сцена"``
   :пример-2: ``"param_1":"Режим"``
   :пример-3: ``"param_1":""``

.. _scene-param_2:
param_2
*****
   Используется для изменения текста в UI-блоке. Например, позволяет уточнить, где будет активирован данный сценарий или режим. Также может использоваться вместо ``param_1``, если вы хотите указать там его значение. Если панель установлена в конкретной комнате, а все сценарии или режимы относятся к ней, это поле можно оставить пустым.

   :имя поля: ``param_2``
   :тип объекта: ``String``
   :наличие значения: Может быть пустым
   :ограничение строки: 11 символов
   :пример-1: ``"param_2":"Сцена"``
   :пример-2: ``"param_2":"Спальня"``
   :пример-3: ``"param_2":""``

.. _scene-param_3:
param_3
*****
   Используется для изменения текста в UI-блоке, чтобы указать название сценария или режима.

   :имя поля: ``param_3``
   :тип объекта: ``String``
   :наличие значения: Обязательно
   :ограничение строки: 11 символов
   :пример-1: ``"param_3":"Ужин"``
   :пример-2: ``"param_3":"Кино"``
   :пример-3: ``"param_3":"Вечер"``

.. _scene-icon:
icon
*****
   Используется для изменения иконки в UI-блоке.

   :имя поля: ``icon``
   :тип объекта: ``String``
   :наличие значения: Обязательно
   :ограничение строки: Стандартный unicode
   :пример-1: ``"icon":"\uDB81\uDF64"`` - иконка 
   :пример-2: ``"icon":"\uDB81\uDF64"`` - иконка
   :пример-3: ``"icon":"\uDB81\uDF64"`` - иконка

.. _scene-command_topic:
command_topic
*****
   Используется для назначения командного MQTT топика, куда будет отправлять указанное сообщение(команда) в ``payload`` при нажатие на блок Сценария или Режима.

   :имя поля: ``command_topic``
   :тип объекта: ``String``
   :наличие значения: Обязательно
   :ограничение строки: нет ограничения
   :пример-1: ``"command_topic":"panel/scenes/1"``
   :пример-2: ``"command_topic":"/devices/panel/controls/scenes_1/on"``
   :пример-3: ``"command_topic":"home/living_room/party_scene/set"``

.. _scene-payload:
payload
*****
   Используется для назначения сообщения(команды) для отправки на командный MQTT топик ``command_topic``, при нажатие на блок Сценария или Режима.

   :имя поля: ``payload``
   :тип объекта: ``String``
   :наличие значения: Обязательно
   :ограничение строки: нет ограничения
   :пример-1: ``"payload":"1"``
   :пример-2: ``"payload":"ON"``
   :пример-3: ``"payload":"morning"``

Примеры
-------

Пример для двух блоков::

    {
        "block": 1,
        "type": "scene",
        "data": 
        {
            "param_1": "Сцена",
            "param_2": "Гостинная",
            "param_3": "Вечеринка",
            "icon": "\uDB84\uDC56",
            "command_topic": "panel/scenes/1",
            "payload": "1"
        }
    },
    {
        "block": 2,
        "type": "scene",
        "data": 
        {
            "param_1": "",
            "param_2": "Режим",
            "param_3": "Чтение",
            "icon": "\uDB84\uDC56",
            "command_topic": "/devices/panel/controls/scenes/on",
            "payload": "reading"
        }
    }
